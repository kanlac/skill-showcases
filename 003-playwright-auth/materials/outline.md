# 文章大纲：Playwright Auth Manager - 让 Agent 自己登录网站

## 一、开篇：痛点切入

**标题建议**：**开发调试切换账号太麻烦？让 Agent 帮你搞定**

- **具体痛点场景**（来自作者亲身经历）：
  - 本地开发需要测试不同用户的展示效果
  - 检查不同权限用户看到的界面是否正确
  - 验证多账号场景下的数据隔离
  - 每次都要：手动登出 → 登录另一个账号 → 测试 → 再切回来
  - 有些网站登录流程复杂（两步验证、验证码），切换一次要好几分钟

- **更广泛的痛点**：
  - Agent 很强，但遇到需要登录的网站就卡壳
  - 想让 Agent 抓取需要登录的数据？先手动复制 Cookie
  - 想让 Agent 测试登录功能？每次都要重新登录

- **解决方案预告**：Playwright Auth Manager 能自动保存和管理登录状态，让 Agent 代替你切换账号、测试功能

## 二、这个 Skill 到底能干什么

- **原理简述**（来自 why.md）：
  - 你登录一次 → 脚本保存认证信息（cookies + localStorage）到 JSON 文件
  - Agent 复用这个 JSON → 自动处于"已登录"状态
  - 就像把家里的钥匙复制了一把交给 Agent

- **核心能力**：
  - 一次配置，持续使用：登录一次，后续自动复用
  - 多账号管理：轻松切换工作账号、测试账号、个人账号
  - Agent 友好集成：与 MCP 深度集成，Agent 自动知道怎么用
  - 安全设计：自动检查 .gitignore，防止泄露

- **典型使用场景**：
  - ✅ 本地开发调试：快速切换不同用户测试功能
  - ✅ 数据抓取：访问需要登录的内部系统、私有项目
  - ✅ 自动化测试：模拟不同权限用户的行为
  - ❌ 生产环境：绝对不行（包含真实凭证）

## 四、拆解：它是如何工作的（来自 skill-analysis.md）

**1. 技术架构（一句话）**
- 基于 Playwright MCP，通过保存和复用浏览器认证状态，让 Coding Agent 像真人一样访问需要登录的网站

**2. 两个核心脚本**

**setup.js - 环境准备**：
- 在 Skill 目录内安装 Playwright，不污染用户项目
- 运行一次即可，安全可重复

**save-auth-state.js - 保存认证**：
- 打开浏览器 → 你手动登录 → 按 Enter → 保存到 JSON
- 文件很小（5-50KB），包含 cookies 和 localStorage

**3. 完整工作流程**
```
1. 运行脚本，手动登录一次
2. 认证信息保存到 JSON 文件（如 user1-auth.json）
3. 配置 MCP Server 加载这个文件
4. 重启 MCP 客户端
5. Agent 自动处于已登录状态 ✅
```

**4. 安全设计**
- 自动检查 .gitignore（防止提交敏感数据）
- 仅用于本地开发（不能上生产）
- 认证文件包含真实凭证（需妥善保管）

## 五、为什么它好用？

**没有这个 Skill 之前**：
- 想让 Agent 访问需要登录的网站？自己写脚本管理 Cookie
- 手动复制 Cookie？只能拿到 Cookie，拿不到 localStorage，格式还容易错
- 每次登录过期？重新手动复制一遍
- 切换账号测试？每次都要登出登入，还要告诉 Agent 怎么用这些认证信息

**有了这个 Skill 之后**：
- 你只需要说："帮我保存这个网站的登录信息"
- Agent 会引导你完成整个流程
- 后续想用？直接说："用账号 A 的身份去做 XXX"
- Agent 自动处于登录状态，无需任何额外操作

**核心价值**：
- ✅ **不需要学习技术**：不懂 Playwright API？没关系，Agent 帮你搞定
- ✅ **不需要写代码**：说出需求即可，Agent 自动引导
- ✅ **不需要操心安全**：自动检查 .gitignore，防止泄露
- ✅ **一次配置，持续使用**：登录一次，永久复用

**市场空白**：目前没有现成的 Skill 做浏览器认证管理，这个 Skill 填补了这个空白。

## 六、如何使用

**就这么简单**：

安装好 Skill 后，直接跟 Agent 说你的需求就行了。

**示例对话**：

```
你：我想保存本地开发网站 localhost:3000 的登录信息

Agent：好的，我来帮你配置。请在新的终端窗口运行以下命令：
    node <path>/save-auth-state.js --url http://localhost:3000 --user dev-account

    运行后：
    1. 浏览器会自动打开
    2. 手动登录你的账号
    3. 登录完成后回到终端按 Enter

    然后我会帮你配置 MCP Server 和 .gitignore
```

```
你：请用账号 A 的身份登录网站，帮我发一个测试帖子

Agent：好的，我会使用账号 A 的认证状态。
    （Agent 自动处于登录状态，直接执行操作）
```

**就是这样**。不需要记命令、不需要学配置，说出需求即可。
