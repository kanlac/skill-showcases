04 拆解

如何实现的？

🎨

关键词：美学 Aesthetic

沉淀了《前端美学：提示词指南》中的内容

查看完整指南 →

可以看到，如果没有指导，Claude 通常会默认生成以白色和紫色为背景的简约设计。有了美学提示后，它会生成更多样化、更具视觉吸引力的设计。

💭 深层思考

大模型虽然有渊博的知识，精通设计原则、字体排印和色彩理论等，但如果不明确要求，它的这些能力都不会得到展现，而你也只能得到中规中矩的结果。

为什么会这样？是模型还不够智能吗？

  • Token 成本：模型厂商不可能无限展开用户请求中的每个关键词，这不划算也没必要。
  • 意图校准：当用户在说「好看」「精美」时，他并不知道自己在说什么，他描述的是一种模糊的感觉。含义需要澄清，意图需要校准。

这也是我做这个系列的初衷——我相信用户没有必要去学习比较那么多 agent/skill。想象一下，当你看到一个案例，第一时间就看到效果，不是夸耀的宣传语，不是含义宽泛的功能介绍，就是真实的评测。

你立马就知道，「我就想要这个类似的」，或者「哦，这个跟我要做的东西还有点差别」。你在查看 showcase 的过程，本质上也是在实行一次潜移默化的意图校准。

06 边界与局限

frontend-design skill 能用来出很好看的前端页面，但它能胜任所有前端开发工作吗？

✅

适用场景

  • UI 原型设计与打磨
  • 页面重新设计
  • 视觉风格探索
  • 快速出图和迭代
⚠️

不适用场景

  • 复杂的业务逻辑
  • 状态管理
  • 后端集成
  • 大型应用架构

建议：主要侧重 UI 设计，建议利用它充分做好原型打磨，重新设计页面也可以，但不要让它做比较重的业务逻辑。如果你在处理状态管理或复杂的业务逻辑，可以使用拥有具体技术栈的开发经验或架构经验的 skill。
