


一个组件设计需求。
```
帮我设计一个电影票选座移动端小程序页面。
业务背景: 用户在购买票务时需要直观地选择座位。场馆结构复杂，座位数量从 30 到 200 不等，且分为不同票价区域。
受众: 普通大众用户（C端）。设备差异大（从千元机到最新 iPhone），手指操作精度不一。
功能需求:
手势操作: 支持双指缩放 (Pinch-to-zoom)、单指拖拽 (Pan)。
迷你导航图 (Mini-map): 当用户放大地图时，左上角出现缩略图，显示当前视窗在整个场馆的位置。
智能吸附: 缩放比例过小时，点击区域模糊，需自动放大并定位到点击的大致区域，而不是直接触发选座，避免误触。
生成在新目录下。
```
